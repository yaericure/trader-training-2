title: "資金管理系統 C 與停損公式"
---


延續上一課系統 B 的侷限,這一課介紹作者實際採用的系統 C——透過拆分成兩筆同時進行的投注,搭配不同寬度的停損,嘗試在放大獲利與控制風險之間取得更細緻的平衡。

## 本章重點速覽

- 系統 C 將每筆交易拆成兩部分:主要部位維持原本較寬的標準停損,加碼部位改用較窄的停損
- 窄停損的加碼部位以當天第一根 5 分鐘 K 線的高低點為基準設定
- 計算加碼投注規模的公式,是把前一筆獲利的一半,除以兩部分停損點數的總和
- 這套系統把交易歸類為勝率較低但報酬較高的類型,改變了原本系統的風險報酬結構
- 實際交易案例顯示,系統 C 在多數獲利月份表現亮眼,但遇到虧損時同樣會同時影響兩個部位

## 核心概念

### 拆分投注的設計邏輯

:::note
系統 C 同時對每筆交易下兩筆投注:第一筆沿用原本的標準停損寬度,確保基本部位維持原有的穩定表現;第二筆則刻意採用較窄的停損,目的是在市場表現順暢的行情中額外爭取更多獲利空間。
:::

作者坦言,這種做法表面上違背了先前提倡的停損應保持寬鬆的原則,但因為窄停損只用於額外加碼的部位,而非全部資金,整體風險仍在可控範圍內。

### 窄停損的設定基準

:::tip
加碼部位的窄停損,是以交易日當天第一根 5 分鐘 K 線的高點或低點為基準,再往外留一段緩衝距離。這根 K 線的長度會隨著每個交易日的市況不同而變化,因此每一筆加碼投注所承擔的風險點數也會逐日調整。
:::

這種設計讓停損寬度能夠貼近當天實際的市場波動狀況,而不是套用一成不變的固定點數,是系統 C 相對系統 B 更精細的地方。

### 加碼投注規模的計算公式

:::tip[進階]
計算下一筆加碼投注規模時,作者的公式是:將前一筆交易獲利除以二,再除以本次交易兩部分停損點數的總和(標準停損加上窄停損),所得結果四捨五入後,做為兩個部位各自的每點投注金額。
:::

這個公式延續系統 B 只拿前次獲利的一半來冒險的精神,但因為多考慮了窄停損部位的風險點數,計算過程比系統 B 更複雜,也更貼近實際交易時面對的雙重風險結構。

## 深入解說

作者透過一整個月的實際交易紀錄示範系統 C 的運作。多數交易日裡,主要部位與加碼部位同時獲利,使得整體績效隨著連續獲利逐步放大投注規模而加速成長。這也印證了系統 C 被歸類為勝率較低、報酬較高類型的原因——當窄停損部位順利獲利時,額外賺取的點數往往相當可觀。

然而,一旦遇到虧損,系統 C 的雙重投注結構會讓虧損同時發生在兩個部位上,金額也會比單一部位的系統更高。作者在案例中呈現了這樣的虧損情境,提醒讀者系統 C 雖然提升了獲利潛力,卻也放大了單次虧損的衝擊,交易者必須有心理準備接受這種風險報酬結構的轉變。

作者最後強調,這整套加碼機制的重點並不在於公式本身有多精巧,而在於交易者是否能夠掌握時機——也就是判斷什麼樣的市況適合積極加碼,什麼樣的市況應該保守收手。市場波動性偏低的時期,即便公式計算出可以加碼,實際上也未必是值得冒險的時機。

:::caution
作者提醒,系統 C 屬於較進階的技術,初學者不宜貿然套用,務必等到完全熟悉基本的資金管理邏輯後,再嘗試這種雙重投注的加碼方式。
:::
